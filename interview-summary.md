# Interview Summary

## Short Project Explanation

This project is a simulated Azure Cloud Security Review lab focused on identity, access control, MFA, Conditional Access, risky sign-ins, privileged access, Microsoft Sentinel logging concepts, and account compromise response.

I created this project to connect my SOC learning with cloud security, because modern cyber security depends heavily on secure cloud environments, identity protection, and good security configuration.

This project demonstrates my understanding of:

- Microsoft Entra ID security concepts
- MFA and Conditional Access
- Privileged account protection
- Risky sign-in investigation
- Identity-based attack risks
- Microsoft Sentinel logging concepts
- Risk documentation
- Security recommendations
- Incident response thinking
- Clear written communication

## How This Links to Cloud Engineering in Consulting

Cloud Engineering in Consulting requires strong foundations in cloud environments, secure configuration, identity, access control, documentation, and client-focused recommendations.

This project shows that I understand how cloud misconfigurations and weak identity controls can create security risk, and how practical recommendations can help improve an organisation's security posture.

Although this is a cloud security project, it also connects to SOC and Cyber Defence because many modern incidents involve identity and cloud activity.

## Interview Answer

I created an Azure Cloud Security Review lab to align my SOC learning with cloud security. The project focuses on Microsoft Entra ID, MFA, Conditional Access, privileged access, risky sign-ins, Microsoft Sentinel logging concepts, and account compromise response.

The aim was to practise how a junior cloud security or consulting analyst might review a small cloud environment, identify common risks, document findings, and provide practical recommendations. It also helped me understand how cloud security connects to modern Cyber Defence and SOC work.
